# 安卓版游戏图标接入说明

这个文件说明两个图标入口：

- App 启动图标：控制手机桌面上显示的游戏图标。
- 主界面图标：控制大厅里的头像、资源、扭蛋、商城、邮件等功能图标。

## 放置位置

### App 启动图标

把正方形 PNG 图标放到：

```text
..\02_素材图\Logo\game_icon.png
```

从安卓项目目录看，也就是：

```text
安卓版\02_素材图\Logo\game_icon.png
```

如果你已经有一个文件叫 `game_icon.ico`，但它其实是 PNG 图片改了后缀，安卓版也会尝试读取：

```text
安卓版\02_素材图\Logo\game_icon.ico
```

推荐优先使用真正的 `game_icon.png`。如果 `game_icon.ico` 是标准 Windows ICO，安卓构建脚本可能无法读取；这种情况请导出一张 PNG 放成 `game_icon.png`。

## 自动生成

运行：

```bat
gradlew.bat assembleDebug
```

构建时会自动检测 `game_icon.png`。如果文件存在，会生成以下尺寸的启动图标：

- `mipmap-mdpi\ic_launcher.png`
- `mipmap-hdpi\ic_launcher.png`
- `mipmap-xhdpi\ic_launcher.png`
- `mipmap-xxhdpi\ic_launcher.png`
- `mipmap-xxxhdpi\ic_launcher.png`

同时会生成对应的 `ic_launcher_round.png`。

## 没放图片时

如果没有放 `game_icon.png`，项目会继续使用默认占位启动图标，不影响 APK 编译。

## 主界面图标映射表

主界面图标不要求你修改原图片名称。把 PNG 图片放到：

```text
安卓版\02_素材图\按钮图标
```

然后打开同目录下的：

```text
主界面图标映射表.csv
```

在第三列 `图片文件名` 填入你的原始图片名，例如：

```csv
功能编号,功能名称,图片文件名
gacha,扭蛋,我的扭蛋图标(最终版).png
shop,商城,商城入口 图标.png
activity,活动大厅,活动大厅.png
```

如果你漏写 `.png` 后缀，构建脚本会自动再尝试补 `.png` 查找，例如 `金币图标` 会继续尝试查找 `金币图标.png`。

支持的功能编号如下：

| 功能编号 | 功能名称 |
| --- | --- |
| avatar_blue | 蓝色头像 |
| avatar_red | 红色头像 |
| avatar_gold | 金色头像 |
| gold | 金币 |
| diamond | 钻石 |
| coupon | 点券 |
| kuroko_ticket | 黑子券 |
| mail | 邮件 |
| market | 篮球交易市场 |
| member_card | 会员卡商店 |
| season | 赛季征程 |
| limited_shop | 限时商店 |
| gacha | 扭蛋 |
| shop | 商城 |
| activity | 活动大厅 |

运行 `gradlew.bat assembleDebug` 时，构建脚本会读取映射表。找到图片就接入主界面；没填写或没找到图片时，对应位置继续使用默认占位图标，不会导致编译失败。
